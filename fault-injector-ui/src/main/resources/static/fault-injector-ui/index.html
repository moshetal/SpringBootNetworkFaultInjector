Quick reference

Recipes for common scenarios.

Match dimensions

  • host-pattern — Java regex on URI.getHost(). Empty / blank means match any.
  • url-pattern — Java regex on the full URL string.
  • methods — optional set; empty means any method.
  • Rules evaluate in declared order. First match wins.

Common regex snippets

api\.billing\..*           # any host under api.billing.*
.*\/orders\/.*             # any URL containing /orders/
^https?:\/\/.*\.flaky\..*  # only flaky.* hosts

Trigger modes

  • PROBABILITY — roll a random number per matching request against probability.
  • EVERY_N — deterministic; fires on every Nth matching request.

Fault types

  • DELAY — sleep for delayMs, then proceed.
  • ERROR — short-circuit with synthetic errorStatus.
  • BOTH — delay, then short-circuit with the error.

curl recipes

curl -s ${HOST}/fault-injector/api/config | jq .

# Toggle a rule
curl -X POST ${HOST}/fault-injector/api/rules/slow-billing/enabled \
     -H 'content-type: application/json' \
     -d '{"enabled": true}'

# Reset all counters
curl -X POST ${HOST}/fault-injector/api/metrics/reset \
     -H 'content-type: application/json' -d '{}'
